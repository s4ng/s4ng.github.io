# s4ng's blog

[Docusaurus](https://docusaurus.io/) 기반 개인 블로그 + 공부 정리 사이트.
`main` 브랜치에 push하면 GitHub Actions가 자동으로 빌드·배포합니다.
배포 주소: https://s4ng.github.io/

## 콘텐츠 구조

- `blog/` — 블로그 글. 파일명 `YYYY-MM-DD-slug.md`, frontmatter에 `title`/`date`/`tags`/`authors`.
- `docs/` — 강의·책 공부 정리. **폴더 계층이 곧 사이드바 계층**입니다.
  - 폴더의 라벨/순서/접힘은 각 폴더의 `_category_.json`으로 제어합니다.
  - 사이드바는 폴더 구조에서 자동 생성(autogenerated)되므로 수동 편집이 필요 없습니다.

## 로컬 개발

```bash
npm install      # 최초 1회 (또는 의존성 변경 시)
npm start        # 개발 서버 (http://localhost:3000)
npm run build    # 프로덕션 빌드 (배포 전 검증)
npm run serve    # 빌드 결과를 로컬에서 확인
```

## 배포

1. (최초 1회) GitHub 저장소 **Settings → Pages → Build and deployment → Source = "GitHub Actions"** 로 설정.
2. `main` 브랜치에 push → `.github/workflows/deploy.yml` 워크플로가 자동 실행되어 GitHub Pages에 배포.

> 빠른 수정은 GitHub 웹 에디터(저장소에서 `.` 키 → github.dev)로도 가능합니다.

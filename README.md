# s4ng's blog

[Docusaurus](https://docusaurus.io/) 기반 개인 블로그 + 공부 정리 사이트.
`main` 브랜치에 push하면 GitHub Actions가 자동으로 빌드·배포합니다.
배포 주소: https://s4ng.github.io/

## 콘텐츠 구조

- `blog/` — 블로그 글. 파일명 `YYYY-MM-DD-slug.md`, frontmatter에 `title`/`date`/`tags`/`authors`.
- `docs/` — 강의·책 공부 정리. **폴더 계층이 곧 사이드바 계층**입니다.
  - 폴더의 라벨/순서/접힘은 각 폴더의 `_category_.json`으로 제어합니다.
  - 사이드바는 폴더 구조에서 자동 생성(autogenerated)되므로 수동 편집이 필요 없습니다.

> 참고: 블로그는 사이트 루트(`/`)에 서빙됩니다. 글 파일은 `blog/`에 두지만 URL은 날짜 기반(`/YYYY/MM/DD/slug`)입니다.

## 글 작성

### 블로그 글

`blog/`에 `YYYY-MM-DD-slug.md` 파일을 만들고 frontmatter를 채웁니다.

```md
---
title: 글 제목
date: 2025-08-14
authors: [s4ng]
tags: [dev]
---

본문 시작...

{/* truncate */}

"더보기" 이후에 이어지는 내용...
```

- `{/* truncate */}` 위쪽이 목록에서 보이는 미리보기입니다.
- ⚠️ `.md`도 MDX로 처리되므로 HTML 주석 `<!-- -->` 대신 `{/* */}`를 쓰세요.
- 본문의 `{ }`·`< >`는 JSX로 오해받아 에러가 날 수 있으니 코드라면 백틱(`` `{x}` ``)으로 감싸세요.

### 공부 정리 (docs)

`docs/` 아래에 폴더를 만들면 그 계층이 곧 사이드바가 됩니다. 폴더마다 `_category_.json`으로 라벨·순서 지정:

```json
{ "label": "운영체제 강의", "position": 1, "collapsed": false }
```

## 이미지 첨부

### 방식 A: 글 옆 폴더 (블로그 글 추천)

글을 폴더로 만들고 이미지를 옆에 둔 뒤 **상대경로**로 참조합니다.

```
blog/
└─ 2025-08-14-my-post/
   ├─ index.md          # 본문 (파일명 index.md 고정)
   └─ screenshot.png
```

```md
![스크린샷](./screenshot.png)
```

### 방식 B: static 폴더 (공용 이미지·로고)

`static/img/`에 두고 **절대경로**(`static` 생략)로 참조합니다.

```md
![프로필](/img/profile.png)
```

- 크기 조절이 필요하면 `<img src="./screenshot.png" width="400" />`처럼 HTML 태그 사용.
- VS Code에선 마크다운에 이미지를 드래그앤드롭하면 경로가 자동 입력됩니다.
- 이미지도 git에 커밋되니 큰 원본은 미리 줄여서 올리세요.

## Notion 글 마이그레이션

1. Notion 페이지/DB 열기 → `···` → **Export**
   - **Format: `Markdown & CSV`**
   - **Include subpages: 켜기**
   - **Create folders for subpages: 켜기** (중첩 구조 유지에 중요)
2. zip 압축 해제 후 저장소 구조로 배치
   - 블로그 글 → `blog/`, 파일명을 `YYYY-MM-DD-slug.md`로 정리하고 frontmatter 추가
   - 공부 정리 → `docs/` 하위 폴더 트리로 재현, 각 폴더에 `_category_.json` 추가
3. 변환 시 보정할 점
   - HTML 주석 `<!-- -->` → `{/* */}`
   - 파일명/폴더명의 공백·해시(`My Page abc123.md`) → 영문 슬러그(`my-page.md`)
   - Notion 내부 링크(`다른글%20abc123.md`) → Docusaurus 상대경로로 수정
   - 이미지 경로를 위 "이미지 첨부" 규칙에 맞게 조정
4. **`npm run build`로 검증** — 깨진 링크·MDX 문법 오류를 빌드가 잡아줍니다.

## 로컬 개발

```bash
npm install      # 최초 1회 (또는 의존성 변경 시)
npm start        # 개발 서버 (http://localhost:3000)
npm run build    # 프로덕션 빌드 (배포 전 검증)
npm run serve    # 빌드 결과를 로컬에서 확인
```

## 배포

1. (최초 1회) GitHub 저장소 **Settings → Pages → Build and deployment → Source = "GitHub Actions"** 로 설정.
2. `main` 브랜치에 push → `.github/workflows/deploy.yml` 워크플로가 자동 실행되어 GitHub Pages에 배포.

> 빠른 수정은 GitHub 웹 에디터(저장소에서 `.` 키 → github.dev)로도 가능합니다.
